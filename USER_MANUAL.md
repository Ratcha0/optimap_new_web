## 1. บทนำและสิทธิ์การเข้าถึง 🚀
ระบบ OptiMap ถูกออกแบบมาเพื่อจัดการงานบริการนอกสถานที่แบบครบวงจร โดยแบ่งสิทธิ์การใช้งานเป็น 3 ระดับหลัก:

### 🕵️ สำหรับแอดมิน (Admin) - "ศูนย์ควบคุมงาน"
*   **Command Center:** ดูภาพรวมการทำงานของช่างทุกคนในจังหวัด
*   **Global Map:** ติดตามตำแหน่งช่างทุกคนแบบเรียลไทม์บนแผนที่เดียว
*   **Telematics Monitoring:** ดูสุขภาพรถของช่าง (ความร้อน, แบตเตอรี่) ได้ทันที

### 👨‍🔧 สำหรับช่าง (Technician) - "มืออาชีพภาคสนาม"
*   **Real-time Navigation:** ระบบนำทางอัจฉริยะพร้อมเสียงแจ้งเตือน
*   **Vehicle Dashboard:** ตรวจสอบสถานะรถยนต์ขณะปฏิบัติงาน
*   **Offline Sync:** ทำงานได้แม้ในจุดอับสัญญาณ ระบบจะซิงค์ข้อมูลให้เมื่อเน็ตกลับมา

### 🏠 สำหรับลูกค้า (Customer) - "ความอุ่นใจในการบริการ"
*   **Live Tracking:** เห็นตำแหน่งช่างเคลื่อนที่มาหาท่านแบบนุ่มนวล (Smooth Movement)
*   **Repair Ticket:** แจ้งซ่อมพร้อมระบุพิกัดที่แม่นยำ (GPS หรือพิกัดบ้าน)

## 2. ฟีเจอร์เด่นระดับพรีเมียม (Premium Features) ✨

### �️ ระบบนำทาง 3D มุมมองสมจริง (3D Navigation Perspective)
*   **Immersive Experience:** มุมมอง 3D แบบลาดเอียง (Tilt) ช่วยให้เห็นเส้นทางข้างหน้าได้ไกลขึ้น และเห็นสิ่งปลูกสร้างในรูปแบบมิติจำลอง
*   **Mobile Optimized:** ใช้เทคนิค CSS Perspective แทนการใช้ WebGL หนักๆ ทำให้มือถือรุ่นทั่วไปสามารถใช้งานโหมด 3D ได้ลื่นไหล
*   **Visual Stretching:** หมุดสถานีและรถช่างถูกปรับสัดส่วน (Stretching) ให้ดู "ตั้งตระหง่าน" บนแผนที่ 3D เพื่อการมองเห็นที่เด่นชัด

### 🛡️ ระบบสถียรภาพสูง (High Reliability & Stability)
*   **Dual-Map Provider:** ระบบมี Fallback อัตโนมัติ หากเซิร์ฟเวอร์แผนที่หลักมีปัญหา ระบบจะสลับไปใช้สำรองทันทีเพื่อให้การนำทางไม่ขัดข้อง
*   **Dead Reckoning Navigation:** เมื่อสัญญาณ GPS หาย (เช่น อยู่ใต้ทางด่วนหรือที่อับสัญญาณ) ระบบจะพยากรณ์ตำแหน่งรถล่วงหน้าตามความเร็วและทิศทางล่าสุด เพื่อให้หมุดรถเคลื่อนที่ต่อเนื่องไม่หยุดนิ่ง
*   **Snap-to-Route:** ตัวรถจะถูกดึงให้ติดกับถนนเสมอ (Map Matching) ป้องกันหมุดไหลออกนอกเส้นทางแม้ค่า GPS จะมีความคลาดเคลื่อน

### 📡 ระบบทำงานออฟไลน์ (Offline & Connection Manager)
*   **Map Prefetching:** ระบบจะทำการดาวน์โหลดแผนที่ล่วงหน้า (Zoom 16-18) ในรัศมีรอบเส้นทางนำทาง เพื่อให้แสดงผลได้แม้ไม่มีเน็ต
*   **Local Storage Sync:** เมื่ออินเทอร์เน็ตขาดหาย ระบบจะแสดงสถานะ **"Warning"** และเก็บข้อมูลพิกัดไว้ในเครื่องชั่วคราว
*   **Auto-Sync:** ข้อมูลจะถูกส่งขึ้นระบบอัตโนมัติทันทีที่กลับมาออนไลน์ เพื่อให้ประวัติการเดินทางไม่ขาดช่วง

### ☁️ การจัดการทรัพยากร (Resource Management - Supabase)
เพื่อให้ระบบรันบนแพ็กเกจฟรีได้ยาวนาน เราใช้เทคนิคพิเศษดังนี้:
*   **Upsert Technology:** ใช้การเขียนทับพิกัดล่าสุดแทนการเพิ่มแถวใหม่ตลอดเวลา ลดการบวมของฐานข้อมูล
*   **Auto-Cleanup (SQL Cron):** ระบบตั้งเวลาลบประวัติการเดินทางที่เก่าเกินกำหนด (เช่น 7-14 วัน) อัตโนมัติ เพื่อคุมขนาดไฟล์ไม่ให้เกิน 500MB
*   **Realtime Optimization:** ปรับจูนการส่งข้อความให้เหมาะสมเพื่อไม่ให้เกินโควต้า 2 ล้านข้อความต่อเดือน

### 🌡️ ระบบตรวจสอบสุขภาพรถ (Vehicle Telematics)
ระบบจะคอยเฝ้าระวังตัวเลขสำคัญ หากอยู่นอกเกณฑ์เลขจะเปลี่ยนเป็น **สีแดงกะพริบ** ทันที:
*   **ความร้อนเครื่องยนต์:** เตือนเมื่อเกิน **105°C**
*   **แรงดันแบตเตอรี่:** เตือนเมื่อต่ำกว่า **13.0V**
*   **ระดับน้ำ/น้ำมัน:** เตือนเมื่อต่ำกว่า **40% / 30%**

### � ระบบทีมและการทำงานร่วมกัน (Team Collaboration)
*   **Multi-Tech Visibility:** ในหน้าจอนำทาง ช่างสามารถเห็นตำแหน่งเพื่อนร่วมทีมที่รับงานเดียวกันได้แบบเรียลไทม์
*   **Team Invitation:** ระบบเชิญเพื่อนร่วมทีมเข้ามาช่วยงาน (Support) พร้อมแชร์ข้อมูลพิกัดและรายละเอียดงานทันที

## 3. คู่มือการใช้งานแต่ละส่วน 📖

### 🕵️ สำหรับ Admin: แผนที่คุมงานรวม
1.  ไปที่เมนู **"แผนที่ติดตาม"**
2.  ท่านจะเห็นหมุดช่างทุกคน:
    *   **จุดสีเขียว:** ออนไลน์และกำลังเคลื่อนที่
    *   **จุดสีเทา:** ออฟไลน์
3.  คลิกที่รูปรถช่างเพื่อดู **"สถานะเครื่องยนต์"** และเบอร์โทรติดต่อช่างรายนั้น

### 🛠️ สำหรับช่าง: การนำทางและอัปเดตงาน
1.  **เริ่มงาน:** เลือกจุดหมายจากพิกัดลูกค้าหรือค้นหาสถานที่
2.  **สลับโหมดมุมมอง:** สามารถเลือกเปิด/ปิดโหมด **3D (🕶️)** ได้ตามความถนัด
    *   *เทคนิค:* ปุ่ม 3D และปุ่มระบุตำแหน่งถูกปรับขนาดให้กะทัดรัด (Compact size) เพื่อไม่ให้บังทัศนวิสัยบนแผนที่
    *   *แนะนำ:* หากใช้มือถือรุ่นเก่ามาก หรือรู้สึกกระตุก ให้ปิดโหมด 3D เพื่อประสิทธิภาพสูงสุดในโหมด 2D
3.  **นิ่งและแม่น:** ใช้ปุ่ม **Auto-Snap (🎯)** เพื่อให้แผนที่หมุนตามทิศหน้ารถอัตโนมัติ
4.  **ประหยัดพลังงาน:** ระบบมี **Wake Lock** ช่วยให้หน้าจอไม่ดับขณะนำทาง แม้ไม่ได้สัมผัสหน้าจอ

### 🤳 สำหรับลูกค้า: การแจ้งเหตุและติดตาม
1.  **แจ้งซ่อม:** กดปุ่ม "แจ้งปัญหา" -> เลือกพิกัดบ้านที่บันทึกไว้ (เร็วที่สุด)
2.  **แชร์พิกัด:** หากต้องการให้เพื่อนช่วยดูช่าง สามารถกดปุ่ม **"🔗 แชร์ตำแหน่ง"** ส่งเข้า LINE ได้ทันที

## 4. ข้อมูลทางเทคนิค (Technical Brief) 👨‍💻
*   **Infrastructure:** รันบนระบบ Cloud ที่มีความสถียรสูง พร้อมระบบ Backup ข้อมูลอัตโนมัติ
*   **Core Logic:**
    *   **Supabase:** จัดการพิกัดแบบ Real-time (Low Latency)
*   **MapLibre GL JS Engine:** ใช้เอนจินการวาดแผนที่แบบ Vector ที่รองรับการทำ 3D เต็มรูปแบบ ลื่นไหล และกินทรัพยากรน้อยกว่าระบบเดิม
*   **Network Resilience:** ระบบทนต่อสัญญาณอินเทอร์เน็ตและการระบุพิกัดที่ไม่เสถียรด้วยกลไก Geolocation Retry และ Local Queueing

## 6. ระบบเซิร์ฟเวอร์นำทางส่วนตัว (Private OSRM Hosting) 🖥️

เพื่อให้โปรเจกต์รองรับช่างเทคนิคจำนวนมาก (Scale) โดยไม่เสียค่าใช้จ่าย API มหาศาล เราจึงใช้ระบบ **Self-hosted OSRM**:

### 📦 ขั้นตอนการติดตั้ง (Deployment Workflow)
1. **Download:** โหลดข้อมูลแผนที่ล่าสุด (`thailand-latest.osm.pbf`)
2. **Extract:** ระบุโปรไฟล์ยานพาหนะ (car.lua) และสกัดข้อมูลถนน
3. **Partition & Customize:** จัดหมวดหมู่สำหรับการหาเส้นทางแบบรวดเร็ว (MLD Algorithm)
4. **Serve:** รัน API Server ผ่าน Docker พอร์ต **5000**

### 🖥️ รายละเอียดสเปกเซิร์ฟเวอร์ (Server Specifications)

| ทรัพยากร (Resource) | ขั้นต่ำ (Minimum) | แนะนำ (Recommended) |
| :--- | :--- | :--- |
| **CPU** | 2 vCPU | **4 vCPU** |
| **RAM** | 4 GB | **8 GB** |
| **Storage (SSD)** | 20 GB | **40 - 50 GB** |
| **Network** | 100 Mbps | **1 Gbps** |
| **OS** | Ubuntu 22.04 LTS | Ubuntu 24.04 LTS |

### 🛠️ การตั้งค่าการเชื่อมต่อ (Connectivity)
*   **Local Testing:** ใช้ `http://localhost:5000` (จำกัดเฉพาะในคอมตัวกลาง)
*   **Mobile Testing (Wi-Fi):** ใช้เลข IP ของเครื่องเซิร์ฟเวอร์ (เช่น `http://192.168.1.xxx:5000`)
*   **Remote Testing (ngrok):** ใช้ URL แปลงพิกัดเพื่อเทสผ่าน 4G/5G จากนอกสถานที่
*   **Production Hosting:** แนะนำให้ใช้ **Google Cloud (GCP)** หรือ **Oracle Cloud (RAM 24GB Free)** เพื่อความเสถียรสูงสุด

*ระบบจะทำการโหลดข้อมูลแผนที่ประเทศไทยทั้งหมด แต่ว่าจะใช้เวลาประมาณ 15-20 นาทีตอนเปิดครั้งแรก และถ้าไม่มีผู้ใช้งาน 2-3 วันระบบจะเข้าสู่สถานะ sleep mode แต่ถ้ามีผู้ใช้งานจะเริ่มทำงานอีกครั้ง แต่จะใช้เวลาประมาณ 15-20 นาทีอีกครั้ง

*เราไม่สามารถ build โหลดครั้งเดียวได้เนื่องจากข้อจำกัดของ hugging face ทำให้ build ไม่เสร็จ ระบบจะไม่สามารถใช้งานได้

# Stage 1: โหลดแผนที่ประเทศไทย (304 MB)
FROM alpine:latest AS downloader
RUN apk add --no-cache curl
WORKDIR /data
RUN curl -fL -A "Mozilla/5.0" -o map.pbf https://download.geofabrik.de/asia/thailand-latest.osm.pbf

# Stage 2: รัน OSRM
FROM osrm/osrm-backend:latest
WORKDIR /osrm-data
COPY --from=downloader /data/map.pbf ./map.pbf

USER root
RUN chown -R 1000:1000 /osrm-data
USER 1000

ENTRYPOINT []
EXPOSE 7860

# --- สั่งรันเรียงลำดับ (ใช้ 1 Thread เพื่อให้แรม 16GB ไม่ล้น) ---
# ขั้นตอนนี้จะใช้เวลาประมาณ 15-20 นาทีในหน้า Logs นะครับ
CMD echo "--- [1/3] EXTRACTING THAILAND (PLEASE WAIT 15 MIN) ---" && \
    osrm-extract -p /opt/car.lua map.pbf --threads 1 && \
    echo "--- [2/3] PREPARING ROUTES (THREADS:1) ---" && \
    osrm-partition map.osrm --threads 1 && \
    osrm-customize map.osrm --threads 1 && \
    echo "--- [3/3] THAILAND SERVER ONLINE! ---" && \
    osrm-routed --algorithm mld map.osrm --port 7860
